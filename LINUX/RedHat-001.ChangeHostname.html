ホスト名の変更¶

サマリー　〜ファイル編〜¶

/etc/sysconfig/network¶

サーバがネットワーク通信に必要な情報が記載されています。


/etc/hosts¶

サーバ自身で内部アクセスが必要である場合には、/etc/hosts ファイルを編集します。


/etc/hostname¶

サーバのホスト名を永続的に変更するには、/etc/hostnameファイル を編集します。


/etc/cloud/cloud.cfg¶

EC2インスタンスでは、永続的なホスト名を設定すために、上述した3つのファイルとは別に/etc/cloud/cloud.cfgファイルを編集する必要があります。 当該ファイルを編集していない場合、再起動後に編集したファイル名がデフォルト値に上書きされてしまいます。



サマリー　〜コマンド編〜¶

では以下のコマンドを扱いします。

catコマンド¶

ファイル内容を確認するコマンドです。 ファイル内容を確認するコマンドとしてはviewコマンドやmoreコマンドなどがあり、 使い勝手はそちらの方が良いですが、さらっとログに残せることもあり、 私はしばしばcatを使用します。

/etc/sysconfig/networkの編集¶

1./etc/sysconfig/ディレクトリへ移動します。

#指定ディレクトリへ移動
cd /etc/sysconfig

#カレントディレクトリを確認
pwd

2./etc/sysconfig/network/ファイルの中身を見てみましょう。

#ファイル内容を確認
cat ./network

3.バックアップを取得します。

#保存先ディレクトリを作成
mkdir reg-ConfigBackup

#保存先ディレクトリへバックアップを作成
cp -a network ./reg-ConfigBackup/network_yyyymmdd

#保存先ディレクトリにバックアップがとれていることを確認
ls -lh ./reg-ConfigBackup/

4.ファイルを編集します。 ファイルの編集にはviエディタを使用します。

vi network
#ファイルが開けましたら「a」キーを押下してください。
#コマンドモードからインサートモードへ移行します。

[編集前]

NETWORKING=yes
NOZEROCONF=yes

[編集後]

NETWORKING=yes
NOZEROCONF=yes
HOSTNAME=ホスト名

編集が完了できたら、編集内容を保存して閉じます。「esc」キーを押下してください。インサートモードからコマンドモードに移行します。 コマンドモードに移行できましたら、「:wq!」と入力し、「Enter」キーを押下してください。


5.編集内容を確認します。

#ファイル内容を確認
cat network

#ファイル内容を比較
diff network ./reg-ConfigBackup/network_yyyymmdd

/etc/hostsの編集¶

1.pingコマンドを実行してみましょう。

ping hostname

2./etc/ディレクトリへ移動します。

#指定ディレクトリへ移動
cd /etc/

#カレントディレクトリを確認
pwd

3./etc/hosts/ファイルの中身を見てみましょう。

#ファイル内容を確認
cat /etc/hosts

4.バックアップを取得します。

#保存先ディレクトリを作成
mkdir reg-ConfigBackup

#保存先ディレクトリへバックアップを作成
cp -a hosts ./reg-ConfigBackup/hosts_yyyymmdd

#保存先ディレクトリにバックアップがとれていることを確認
ls -lh ./reg-ConfigBackup/

5.ファイルを編集します。 ファイルの編集にはviエディタを使用します。

vi hosts
#ファイルが開けましたら「a」キーを押下してください。
#コマンドモードからインサートモードへ移行します。

[編集前]

127.0.0.1   localhost localhost.localdomain localhost4 localhost4.localdomain4
::1 localhost localhost.localdomain localhost6 localhost6.localdomain6

[編集後]

127.0.0.1 ホスト名 ホスト名.localdomain localhost4 localhost4.localdomain4
::1 localhost localhost.localdomain localhost6 localhost6.localdomain6

5.編集内容を確認します。

#ファイル内容を確認
cat

#ファイル内容を比較
diff hosts ./reg-ConfigBackup/hosts_yyyymmdd

編集が完了できたら、編集内容を保存して閉じます。「esc」キーを押下してください。インサートモードからコマンドモードに移行します。 コマンドモードに移行できましたら、「:wq!」と入力し、「Enter」キーを押下してください。

5.編集内容を確認します。

diff hostname ./reg-ConfigBackup/hostname_yyyymmdd




/etc/cloud/cloud.cfgの編集¶

1./etc/cloud/ディレクトリへ移動します。

#指定ディレクトリへ移動
cd /etc/cloud/

#カレントディレクトリを確
pwd

2./etc/cloud/cloud.cfgファイルの中身を見てみましょう。

#ファイル内容を確認
cat /etc/cloud/cloud.cfg

3.バックアップを取得します。

#保存先ディレクトリ作成
mkdir reg-ConfigBackup

#保存先ディレクトリへバックアップを作成
cp -a cloud.cfg ./reg-ConfigBackup/cloud.cfg_yyyymmdd

#保存先ディレクトリにバックアップがとれていることを確認
ls -lh ./reg-ConfigBackup/

4.ファイルを編集します。 ファイルの編集にはviエディタを使用します。

vi cloud.cfg
#ファイルが開けましたら「a」キーを押下してください。
#コマンドモードからインサートモードへ移行します。

[編集後]

#末尾に以下追加
preserve_hostname: true

編集が完了できたら、編集内容を保存して閉じます。「esc」キーを押下してください。インサートモードからコマンドモードに移行します。 コマンドモードに移行できましたら、「:wq!」と入力し、「Enter」キーを押下してください。




4.編集内容を確認します。

#ファイル内容を確認
cat cloud.cfg

#保存先ディレクトリにバックアップを作成
diff cloud.cfg ./reg-ConfigBackup/cloud.dfg_yyyymmdd
